Project Overview

This project involves creating a simplified version 1 (v1) of the Byte Dashboard tenant management system. The goal is to rebuild the application with a modern Next.js 16 architecture while preserving core functionality. The new v1 will focus on essential capabilities: contact/people management, application tracking, task management with a kanban board, calendar view, dashboard with key metrics, and basic settings.

Scope of Work

The developer will design and develop a web-based dashboard application using Next.js 16 that includes: • Next.js 16 App Router architecture with React 19 and TypeScript • Clerk authentication system with role-based access • Neon PostgreSQL database with core tables • Contact/People management module with full CRUD operations • Application tracking module with basic status workflow • Task management with drag-and-drop kanban board • Dashboard with statistics, activity feed, and quick actions • Calendar view for tasks and deadlines • Basic settings pages (general, users, billing)

Explicitly Out of Scope for v1
The following features will NOT be included in v1: Visual workflow builder, dynamic form builder, field-level encryption, third-party integrations, AI assistant, custom fields system, advanced audit logging, file attachments, email notifications, CSV import/export, real-time collaboration features, production deployment, mobile-responsive design optimization, support/contact system.

Technical Architecture

Frontend Stack: • Next.js 16.1.6 with App Router and React Server Components • React 19.2.3 with TypeScript 5 • Tailwind CSS 4 for styling • shadcn/ui + Radix UI component library • TanStack Query for server state management Backend & Infrastructure: • Clerk for authentication and user management • Neon PostgreSQL for database • Railway for API and edge functions • Vercel for frontend hosting

Deliverables & Acceptance Criteria

ID Deliverable
P1-A
Foundation Setup & Authentication
Next.js 16 project initialized. Clerk authentication integrated. Users can sign up, sign in, and manage profile. Basic role system functional (Admin, Member, Viewer). Layout with sidebar navigation responsive on all devices.
P1-B
Database Schema & Base Components
Neon database configured with 6-8 core tables. Database schema documented. Base UI components library established (24-30 components). Data table component functional with sorting and filtering.
P2-A
Contacts/People Module
Contact list view with data table working. Contact detail page displays all information. Add/edit contact forms functional. Basic search and filter by name, email, status working. Contact deletion with confirmation prompt. All CRUD operations persist to database correctly.
P2-B
Applications Module
Application list view with status filters. Application detail page shows linked contact and all fields. Add/edit application forms functional. Simple status workflow (Draft → Submitted → Review → Approved/Rejected) working. Applications linked to contacts correctly.
P3-A
Tasks & My Work Kanban Board
Kanban board displays tasks in columns by status. Drag-and-drop to change task status functional. Add/edit/delete tasks working. Tasks can be assigned to users. Task filtering by assignee functional. All task data persists correctly.
P3-B
Dashboard & Analytics
Dashboard displays key metrics (total contacts, applications, tasks). Recent activity feed shows latest actions. Quick action buttons work correctly. Statistics update in real-time when data changes. Charts display data accurately (if included).
P4-A
Calendar & Settings
Calendar view displays tasks by due date. Month/week views functional. Settings pages (general, users, billing) accessible to admins. User management allows invite/edit/remove. General settings persist correctly.
P4-B
Testing & Code Documentation
All CRUD operations tested and verified. Core user flows functional (authentication, contact creation, application tracking, task management). Code documented with comments where necessary. README provided with setup instructions.
